# 资源加载系统使用指南

## 🎯 核心设计

**简洁、无状态、工具化**：
- ✅ 只负责加载和解析，不存储数据
- ✅ 数据通过回调直接传递给业务层
- ✅ 业务层自己决定如何存储和管理数据

## 🚀 基本使用

### 方式1：链式配置（推荐）

```csharp
using Hotfix.ResourceLoader;

// 在进入 Procedure 前配置
GameLoadProcedureV3.Configure()
    .Add("Scene", "BattleScene", (data) =>
    {
        Debug.Log("场景加载完成");
    })
    .Add<byte[]>("Binary", "AnimationDatabase.bytes", (bytes) =>
    {
        // 业务层自己处理数据
        AnimationDatabaseLoader.LoadDatabase("AnimationDatabase.bytes");
    })
    .Add<ConfigData>("Json<ConfigData>", "Config.json", (config) =>
    {
        // 业务层自己存储
        ConfigManager.Instance.SetConfig(config);
    });

// 切换到加载流程
ChangeState<GameLoadProcedureV3>(procedureOwner);
```

### 方式2：使用配置对象

```csharp
var config = new ResourceLoadConfig();
config.Add("Scene", "BattleScene", OnSceneLoaded);
config.Add<byte[]>("Binary", "Data.bytes", OnDataLoaded);

GameLoadProcedureV3.ConfigureLoad(config);
ChangeState<GameLoadProcedureV3>(procedureOwner);
```

## 📝 完整示例

### 示例1：动画系统初始化

```csharp
using Hotfix.ResourceLoader;
using Hotfix.Animation;

public class GameInitializer
{
    public void Initialize()
    {
        GameLoadProcedureV3.Configure()
            .Add<byte[]>("Binary", "AnimationDatabase.bytes", (bytes) =>
            {
                // 业务层处理：加载动画数据库
                AnimationDatabaseSO databaseSO = Resources.Load<AnimationDatabaseSO>("AnimationDatabase");
                if (databaseSO != null)
                {
                    AnimationDatabaseLoader.LoadDatabase("AnimationDatabase.bytes");
                }
            });
    }
}
```

### 示例2：配置系统

```csharp
public class ConfigManager
{
    private GameConfig _config;
    
    public void Initialize()
    {
        GameLoadProcedureV3.Configure()
            .Add<GameConfig>("Json<GameConfig>", "GameConfig.json", 
                (config) =>
                {
                    // 业务层自己存储
                    _config = config;
                    ApplyConfig(config);
                },
                (error) =>
                {
                    Debug.LogError($"配置加载失败: {error}");
                    _config = GetDefaultConfig();
                });
    }
}
```

### 示例3：批量加载

```csharp
GameLoadProcedureV3.Configure()
    .AddBatch(
        "Scene:BattleScene#Binary:Data.bytes#Json<Config>:Config.json",
        (name, data) =>
        {
            Debug.Log($"资源加载成功: {name}");
            // 根据名称处理不同的资源
        },
        (name, error) =>
        {
            Debug.LogError($"资源加载失败: {name}, 错误: {error}");
        }
    );
```

## 📊 数据流

```
业务层配置加载项
    ↓
GameLoadProcedureV3.Configure()
    ↓
SimpleResourceLoader 加载资源
    ↓
加载器解析数据
    ↓
通过回调直接传递给业务层
    ↓
业务层自己存储和管理数据
```

## 💡 最佳实践

1. **在初始化时配置**：在游戏启动时配置加载项
2. **业务层自己存储**：不要依赖加载器存储数据
3. **错误处理**：每个加载项都应该有错误处理
4. **类型安全**：使用泛型方法确保类型安全
