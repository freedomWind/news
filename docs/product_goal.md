# 产品目标

## 背景

本项目目标是构建一个基于 Unity 的移动端信息资讯界面框架，用于承载棋类资讯、赛事内容、对局推荐、棋盘预览和后续互动模块。

界面形态参考 `设计图/首页.jpeg`：以移动端竖屏为主，包含顶部导航、首页信息流、重点对局卡片、赛事快讯和底部 Tab。

## 核心目标

首版目标不是做完整资讯 App，而是先搭建一套可持续扩展的 Unity UI 中间层：

- 使用 `uGUI + TextMeshPro` 作为运行时 UI 主技术栈。
- 支持信息流页面的结构化渲染。
- 支持图文内容、棋局预览、赛事卡片等内容块混排。
- 让页面内容可以由结构化数据驱动，而不是全部写死在 Unity Prefab 中。
- 为后续接入真实接口、文章详情、对局详情、棋盘互动打基础。

## 产品形态

首版聚焦以下页面和能力：

- 首页
- 顶部栏
- 底部 TabBar
- 可滚动内容流
- 重点对局卡片
- 赛事快讯列表
- 棋盘/局面预览占位组件

后续扩展方向：

- 文章详情页
- 对局详情页
- 棋谱回放
- 赛事页
- 数据页
- 我的页
- 图片缓存
- 远程数据接口
- 运营配置页面

## 设计原则

### 内容数据驱动

页面应尽量通过结构化内容数据生成。新增资讯内容、活动卡片、棋局卡片时，应优先扩展 block schema 和 block view，而不是直接修改页面主体逻辑。

### 框架小，协议稳

首版不要追求大而全的 UI 框架。需要优先稳定的是：

- 内容块协议
- 页面和组件分层
- 基础组件命名和职责
- 数据到 UI 的映射规则

### 优先服务棋类资讯场景

通用资讯能力只做到够用。棋盘预览、对局入口、赛事信息、棋谱内容是本项目相对普通资讯 App 的核心差异。

## 首版不做

首版明确不做以下内容：

- 完整 HTML 渲染器
- WebView 图文混排
- 真实后端接口
- 完整棋类规则引擎
- 完整棋谱回放系统
- 复杂动画系统
- 可视化页面搭建器
- 多语言系统
- 支付、登录、推送、分享等原生能力

## 成功标准

首版成功的标志是：

- 能在 Unity 中打开首页 Demo。
- 首页视觉结构接近设计图。
- 首页内容由 mock 数据驱动。
- 至少支持 `featured_match`、`board_preview`、`section_title`、`news_item` 这几类 block。
- 新增一种 block 不需要重写首页主体逻辑。
- 后续可以继续沿同一套文档和计划自动推进。
